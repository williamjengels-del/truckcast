# Marketing screenshots

Product screenshots used by the homepage Phase 2.5 section
("See what operators see"). Generated by
`scripts/capture-screenshots.mjs` against the dedicated demo operator
account (`demo-screenshots@vendcast.co`).

## Files

- `todays-event.png` — dashboard, day-of-event card with weather + contact + sales pace
- `inquiry-inbox.png` — `/dashboard/inquiries` with 4 demo inquiries, engagement signal visible
- `forecast-card.png` — forecast card on a future event with weather adjustment

## How to regenerate

1. Make sure the demo account exists in production Supabase Auth and has loaded sample data via `/api/sample-data/seed`.
2. Set env vars:
   ```
   DEMO_SCREENSHOT_EMAIL=demo-screenshots@vendcast.co
   DEMO_SCREENSHOT_PASSWORD=<password set in Supabase>
   ```
3. Run from the `truckcast/` directory:
   ```
   node scripts/capture-screenshots.mjs
   ```
4. Verify the three PNGs in this directory render correctly on a 1440px-wide screen.
5. Commit the regenerated PNGs.

## Frame chrome

Drop shadow + rounded border are applied by the homepage CSS, not baked into the PNGs themselves. Captures should be tight crops of the relevant UI surface — no browser chrome, no full-page bleed.
